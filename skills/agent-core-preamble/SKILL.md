## Перед работой: проверь rules проекта

Rules — обязательные правила проекта, игнорировать нельзя.

1. Выполни `ls .claude/rules/ ~/.claude/rules/ 2>/dev/null`
   - `.claude/rules/` — проектные (приоритетнее при конфликте)
   - `~/.claude/rules/` — глобальные
2. По названиям файлов определи, какие релевантны твоей задаче — **с запасом**:
   если сомневаешься, читай. Лучше прочесть лишнее, чем пропустить важное.
3. Прочитай выбранные файлы через Read tool. Применяй при работе.

## Проблемы флоу

Если при работе ты столкнулся с проблемой самого процесса — **противоречие двух инструкций, обещанный но недоступный инструмент, невыполнимый шаг, неожиданное поведение харнеса** — задокументируй её двумя действиями:

1. Допиши строку в общий журнал через Bash:
   ```bash
   mkdir -p ~/.claude/projects/$(printf '%s' "$HOME/.claude" | sed 's|[/.]|-|g')/memory
   echo "<YYYY-MM-DD> | <имя-агента> | <repo или cwd> | <одно предложение>" >> ~/.claude/projects/$(printf '%s' "$HOME/.claude" | sed 's|[/.]|-|g')/memory/flow-feedback.md
   ```
2. Продублируй в финальном reply отдельной строкой: `flow-issue: <то же предложение>`.

Это канал только для реальных препятствий, с которыми ты фактически столкнулся в этой задаче. НЕ для улучшательства («промт можно сделать лучше», «я бы переформулировал») — такое не пиши.
