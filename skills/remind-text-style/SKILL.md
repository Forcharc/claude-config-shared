# Правила оформления текста

Два уровня правил с разной применимостью.

## Группа α: Маскировка символов

**Применяется ВЕЗДЕ где пишу я.** Не палься на типичные ИИ-маркеры — люди их не ставят. Касается:
- Коммитов (title и body)
- МР-комментов и ответов коллегам
- Описаний тикетов, постов
- Документации (README, ADR, тех. доки)
- Технических комментов в коде
- Любого текста который читает человек

**НЕ применяется** только внутри настоящих ``` блоков кода ``` — там символы значимы.

### Замены

1. `—` (длинное тире) → `-` (дефис)
2. `…` (символ многоточия) → `...` (три точки)
3. `«» " " ' '` («умные» кавычки и ёлочки) → `""` или `''` (прямые)
4. Неразрывные пробелы → обычные пробелы
5. `→` (стрелка) → `->`
6. **Никаких backticks** вокруг кода, имён файлов, идентификаторов, путей, строк кода. Люди их не ставят — сразу палит ИИ. Пиши `handleAuth()` как handleAuth(), `foo.kt:42` как foo.kt:42. Если собиралась обернуть идентификатор одиночным backtick'ом — просто не оборачивай.

## Группа β: Стиль

**Применяется ТОЛЬКО к коротким интерактивным текстам наружу** — где аудитория ждёт лаконичности:
- МР-комменты
- Ответы коллегам в чатах (Slack, Telegram, тикеты)
- Короткие посты, реплики

**НЕ применяется** к:
- Коммит body — там полезно подробно объяснить «почему сделано»
- Документация (README, ADR, design docs) — там нужны полные объяснения
- Технические комменты в коде — там нужно описать неочевидное полностью

### Правила

- **Короче.** Не разжёвывай очевидное. Адресат не тупой — он знает свой код и свой язык.
- Не описывай, что делает код, который ты комментируешь. Автор это знает.
- Не объясняй механику того, что адресат и так понимает.
- **Один факт + один фикс.** Без преамбул «в этом месте есть проблема...», «обрати внимание...», «смотри...».
- **Идеальный коммент к коду: 1-2 строки.** Указал место, указал фикс. Всё.

## Когда вызывать

- Перед написанием любого текста наружу — обе группы (α + β)
- Перед коммит-сообщением — только группа α (маскировка)
- Перед документацией / ADR / тех. комментами — только группа α
- В рамках `/txt`, `/review-mr`, `/commit` — skill ссылается, применяй автоматом
