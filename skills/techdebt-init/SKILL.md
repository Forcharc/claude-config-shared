Инициализировать структуру хранения техдолга.

## Шаг 0: Определить путь хранения

Алгоритм определения базового пути (с учётом `techdebt_location` и особого случая `~/.claude/`) — см. канонический источник: **`~/.claude/skills/techdebt/SKILL.md` шаг 0**. Используй тот же путь здесь.

## Создать структуру

Создай директорию `<базовый>` и два файла:

### 1. <базовый>/INDEX.md

```markdown
# Tech Debt Index

<!-- Формат записи: - [td-NNN](td-NNN-slug.md) ПРИОРИТЕТ `тег` Краткое описание -->
<!-- Записи группируются по модулям проекта. Пустые секции можно удалять. -->
```

Добавь секции `## модуль/` для каждого модуля верхнего уровня в проекте (определи по структуре директорий). Добавь секцию `## Корневой уровень / инфра` в конце.

### 2. <базовый>/TEMPLATE.md

```markdown
---
id: td-NNN
title: Краткое описание проблемы
module: имя-модуля
tags: [duplication|performance|reliability|security|design|cleanup|testing]
priority: "🔴|🟠|🟡|🟢"
found: YYYY-MM-DD
task: "контекст задачи, при которой нашли"
status: open
originSessionId: "UUID текущей Claude Code сессии (bash ~/.claude/skills/techdebt/session-id.sh)"
---

## Проблема

Что именно не так. Описать конкретно — что происходит, почему это плохо.

## Где

- `module/path/to/File.kt:LINE` — что именно

## Предлагаемое решение

Как можно исправить (если есть идея). Можно несколько вариантов.

## Риски если не исправить

Что случится если оставить как есть.
```

## После создания

Скажи пользователю что структура создана и можно записывать техдолг через `/techdebt`.

При `repo`-режиме — файлы нужно будет закоммитить. При `local`-режиме — коммит не нужен (файлы вне репо).
