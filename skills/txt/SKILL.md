# TXT — вывод информации в текстовый файл

Напиши запрошенную информацию в txt файл и открой его для копирования.

## Аргументы
- `$ARGUMENTS` — что нужно написать в файл

## Примеры запросов
```
/txt описание для МР
/txt список изменений в этой ветке
/txt инструкция по запуску
```

## Алгоритм

1. Сформируй текст на основе запроса и текущего контекста разговора
2. Придумай короткий slug из запроса (kebab-case, латиница):
   - «описание для МР» -> `mr-description`
   - «ответ коллеге» -> `reply`
   - «список изменений» -> `changes`
   - «инструкция по запуску» -> `run-instructions`
   - Если запрос пустой / неслагуемый / одни цифры / спецсимволы — используй `output`
3. Получи timestamp: `Bash: date +%Y%m%d-%H%M%S` — запомни вывод (например, `20260413-171542`)
4. Создай папку: `Bash: mkdir -p .tmp` (идемпотентно)
5. Собери путь: `.tmp/claude-txt-<slug>-<timestamp>.txt`
6. Запиши файл через встроенный **Write** tool (литеральный путь)
7. Открой файл: `Bash: open <путь>`
8. Сообщи пользователю путь к файлу

## Уникальность имени
- Каждый `/txt` создаёт новый файл, предыдущие сохраняются
- НЕ используй фиксированное имя типа `.tmp/claude-txt-output.txt`

## Важно
- Пиши чистый текст без лишних обёрток — пользователь будет копировать содержимое
- Язык текста определяй по контексту (если МР описание — на русском, если код — как уместно)
- Не добавляй заголовки вроде "Описание для МР:" — сразу контент

## Правила оформления текста

См. `/remind-text-style` — маскировка символов (нет длинных тире / умных кавычек / backticks) + стиль (короче, без преамбул, один факт + один фикс).

Применяй эти правила к тексту, который записываешь в файл — пользователь будет копировать и пересылать наружу.
